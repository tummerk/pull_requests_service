В папке service располагаются сервисы - функционал, реализующий основную бизнес-логику  
Они же usecase  
Оперируют доменными сущностями (entities)  
В данном примере подразумевается, что каждый сервис располагается в отдельной папке  
Однако, если сервисов мало, дополнительные папки можно не вводить  
Это не принципиальный момент, нужно смотреть на общее количество файлов и как удобнее структурировать код  
В зависимости от наличия вложенных папок будет меняться и нейминг  
  
Для общей папки service  
Код каждого сервиса будет располагаться в отдельном файле, например - example.go, пакет - service  
Имя типа для сервиса - Example, имя "конструктора" - NewExample  

Для вложенных папок  
Код каждого сервиса будет располагаться в файлах отдельной папки, например - example/service.go, пакет - example  
Имя типа для сервиса - Service, имя "конструктора" - NewService
